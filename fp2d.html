FP2D

This is a project the I worked on during my Senior year at the University of Illinois along with 9 other students. It is a web based application for Professor Nakamura in the Department of Food Science and Human Nutrition. It was developed with the goal of allowing dietitians to help their clients better understand their fiber and protein intake per Calorie in order to form healthier eating habits.

This application has 3 sets of users. The administrator is capable of adding dieticians to the database, as well as updating the applications copy of the USDA database. The dieticians are capable of adding clients, as well as viewing their clients food intake information. The clients are able to log in and enter the foods that they consumed that day. This information is stored in a database and displayed in a graphical representation allowing them to quickly see which foods are healthy or not healthy for them. Using all this information the dieticians are able to set goals for their clients to help them with weight loss goals.

We used multiple technologies to build this application. The backend is written in Ruby on Rails which allowed us to authenticate our users and to block access to any portion of the application that the user shouldn't have access to. The main graph was written using jQuery and jqPlot. All database queries were made using AJAX. This project allowed me the opportunity to see how multiple technologies can be used together to create extensive applications.
